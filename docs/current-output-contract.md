# Current Output Contract

This document catalogs the current schemas and data structures produced by the audit engine.

---

## 1. Class Data Contracts

### `CheckResult` (Python Dataclass)
Defined in `checks/base.py`. Holds metadata and status for individual audit rules.

```python
@dataclass
class CheckResult:
    check_id: str
    check_name: str
    category: str
    severity: Severity # Enum: critical, high, medium, low, info
    passed: bool
    score: int         # 0 (fail) to 100 (pass)
    detail: str        # Summary of finding details
    recommendation: str = ""
    fix_code: Optional[str] = None
    fix_difficulty: str = ""
    impact_estimate: str = ""
    data: dict = field(default_factory=dict)
```

## 2. API / JSON Output Contracts

### A. Grader CLI JSON Output Format
Generated when executing `grader.py -j/--json output.json`.

```json
{
  "url": "https://example.com",
  "score": {
    "overall_score": 85,
    "grade": "B",
    "categories": {
      "Technical SEO": {
        "score": 90,
        "weight": 25,
        "checks_passed": 15,

  "checks": [
    {
      "check_id": "tech_meta_title",

      "score": 100,

```
